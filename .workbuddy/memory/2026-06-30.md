## Pi TUI Crash Fix: CJK-safe visible-width truncation

**Bug**: Pi crashed with `Rendered line 39 exceeds terminal width (219 > 201)` because:
1. `measureVisibleWidth` fallback used `stripAnsi(text).length` — doesn't count CJK as 2 columns
2. `renderCcReviewLogEntry` used `prefix.length` (JS string length, not visible width) to budget message wrap — prefix icons (`ℹ`) and Chinese source names can be wide chars
3. `getTruncateToWidth()` looked for `piTui.truncateToWidth` which pi-tui never exported — was always null, always fell back to broken plain-text truncation

**Fix in `.pi/extensions/cc-review.ts`**:
- `measureVisibleWidth`: replaced naive `length` fallback with full Unicode East-Asian width table (CJK, Hangul, Hiragana, Katakana, Fullwidth, etc.)
- `getTruncateToWidth`: synthesises a width-aware truncator from `piTui.visibleWidth` when `truncateToWidth` is absent
- `truncateWidgetLine`: fallback now uses `measureVisibleWidth` + CJK-aware slice, not `truncateForWidget`
- `renderCcReviewLogEntry`: replaced `prefix.length` with `measureVisibleWidth(prefix)` (also fixes `continuationPrefix` indent)
- `buildCcReviewWidgetLines` live-log loop: added defensive `safeLine = truncateWidgetLine(line, width - 3)` before ANSI wrapping
- Updated `cc-review-static.test.mjs` to match new `prefixVisibleWidth` variable name

All 212 tests pass (64 static + 148 behavior).

## CJK-width truncation refactoring (quality follow-up)

Reviewed previous crash fix against "最优方法" standard. Found 3 quality issues and resolved:
1. **Duplicated CJK width logic**: `measureVisibleWidth` had full EAW table, but two truncation functions used bare `> 0x2E7F` heuristic → **Extracted `charWidth()` as single source of truth** (now used by `countColumns`, `truncatePlainByColumn`, `truncateAnsiByColumn`)
2. **ANSI code loss**: `buildVisibleWidthTruncate` sliced stripped text, losing ANSI colour codes → **New `truncateAnsiByColumn`** walks raw string preserving escape sequences
3. **Code duplication**: `buildVisibleWidthTruncate` and `truncateWidgetLine` fallback both had identical truncation loops → **Single `truncatePlainByColumn`** shared by both
4. **Emoji width bug**: `📄` (U+1F4C4) fell outside EAW ranges → Added `code > 0xFFFF → width 2` as first check (supplementary-plane rule, matches terminal behaviour)

All 212 tests pass (64 static + 148 behavior). Original crash scenario verified: max width 200 ≤ 201.
